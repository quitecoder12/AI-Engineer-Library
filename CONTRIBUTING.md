# Contributing

Welcome to the AI Engineer Library.

Thank you for your interest in contributing.

---

## Mission

Create the world's most practical AI Engineering documentation.

Every contribution should improve:

- Accuracy
- Clarity
- Practicality
- Maintainability

---

## Documentation Principles

- Explain concepts before syntax.
- Use production-ready code.
- Prefer modern Python.
- Include AI-related examples whenever possible.
- Keep examples simple and focused.

---

## Style

- Markdown only
- ATX headings (`#`)
- Sentence case headings
- Wrap lines around 100 characters when practical
- Use fenced code blocks with language identifiers

Example:

```python
print("Hello, AI Engineer!")
```

---

## Folder Structure

```
docs/

python/

numpy/

pandas/

llms/

rag/

projects/

cheatsheets/
```

---

## Commit Messages

Good examples:

```
Add Python variables chapter

Add RAG overview

Improve FastAPI examples

Fix typo in NumPy chapter
```

Avoid generic messages like:

```
Update

Fix

Changes
```

---

## Pull Requests

Every PR should:

- Build successfully
- Follow the style guide
- Include clear explanations
- Be reviewed before merging
